# LifeOS Plans Index

This folder is the single source of truth for project planning and execution tracking.

## Documents

1. `PROJECT_MASTER_PLAN.md`
   - Strategic roadmap, phase goals, architecture guardrails, and delivery sequence.

2. `DEVELOPMENT_TRACKER.md`
   - Live execution tracker with progress tables, completed increments, and next actions.

3. `PHASE_BACKLOG.md`
   - Prioritized backlog for upcoming implementation work.

4. `RELEASE_READINESS_CHECKLIST.md`
   - Operational and technical checks before merge/release.

5. `COPILOT_AUTOPILOT_WORKFLOW.md`
   - Team workflow for using GitHub Copilot Chat/agents safely and efficiently.

6. `BRANCH_PROTECTION_CHECKLIST.md`
   - Repository branch-protection baseline for `main` (required checks/review gates).

7. `GIT_PUBLISH_RUNBOOK.md`
   - Step-by-step process to connect remote, publish `main`, and trigger releases.

## Usage

- Update `DEVELOPMENT_TRACKER.md` after each implementation session.
- Keep `PHASE_BACKLOG.md` prioritized top-to-bottom.
- Use `RELEASE_READINESS_CHECKLIST.md` before PR merge and before pushing to `main`.
- Keep docs concise but always executable (commands, files, acceptance criteria).
